# SitMan Web Apple 风格主题改造设计

## 目标

在不改动现有功能逻辑的前提下，重做单页应用的视觉样式，使其从当前的深色重色块风格切换为更接近苹果系的浅色、玻璃感、柔和层级风格，并支持预设色板切换。

## 范围

- 保留现有单页结构和所有功能行为
- 保留已完成的移动端适配
- 改造标题栏、三栏面板、控制按钮、输入区、笔记卡片、弹窗和滚动区域样式
- 新增预设主题切换入口
- 主题切换结果写入 `localStorage`

## 方案选择

采用“苹果风格 + 预设切换”的方案。

- 默认主题使用浅色玻璃感风格
- 同时提供 4 套预设色板
- 切换时不仅替换主色，还同步调整卡片高光、边框、阴影和强调色

不采用自由配色，避免单页工具界面失控，也减少后续维护成本。

## 视觉设计

### 默认气质

- 页面背景使用浅灰到冷白的柔和渐变
- 主面板改成半透明白色卡片
- 卡片使用较大的圆角和轻阴影
- 使用浅灰描边和高光边界增强层次
- 主强调色使用苹果系蓝色

### 预设色板

- `经典蓝`
- `薄荷绿`
- `石墨灰`
- `暖沙金`

所有主题共享同一套布局与视觉层级，只改变以下变量：

- 主强调色
- 强调色弱化背景
- 边框色
- 阴影色偏向
- 选中态和高亮态

### 样式重点

- 标题栏从纯深底改成浅色玻璃条
- 左右侧栏与中间主区统一成卡片系统
- 按钮分为默认、主按钮、辅助按钮三类
- 输入框和滑条统一成浅色表单风格
- 笔记块从黑底改成白卡叠层结构
- 弹窗使用更大的圆角和毛玻璃背景

## 交互设计

### 主题切换入口

- 放在标题栏右侧
- 采用轻量的胶囊按钮或小圆点色板
- 点击即切换，无需额外确认

### 持久化

- 当前主题写入 `localStorage`
- 页面加载时优先恢复上次选择
- 如未命中则回退到默认主题

## 技术设计

### CSS 变量

引入一套统一主题变量，包括但不限于：

- `--bg-start`
- `--bg-end`
- `--surface`
- `--surface-strong`
- `--surface-soft`
- `--text-primary`
- `--text-secondary`
- `--accent`
- `--accent-soft`
- `--border`
- `--shadow`

现有硬编码颜色逐步替换为变量，确保主题切换覆盖全局。

### 主题挂载

- 在 `body` 上通过 `data-theme` 或类名切换主题
- 用一小段原生 JavaScript 维护主题应用和持久化

## 验收标准

- 页面整体风格明显从深色重色块转为苹果系浅色风格
- 可在页面内切换 4 套预设色板
- 切换后主要区域颜色、边框、按钮和高亮态同步变化
- 刷新页面后仍保留上次主题
- 手机端布局不退化，仍可正常操作

## 风险与约束

- 单文件页面中 CSS 和 JS 耦合较高，改造时只调整样式层，不动业务逻辑
- 苹果风格以“借鉴气质”为目标，不直接复制原生系统 UI 组件
- 保持移动端触控尺寸和当前自适应规则，避免视觉改造破坏已完成的手机适配
